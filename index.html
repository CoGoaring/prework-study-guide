HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • We can use list elements to order information in one of 3 ways, using ul (unordered), li (list items), ol (ordered).
  • The footer element will conatin anything we need only at the very end of our website.
  • In my Git section you can see I added a link using a href inside of one of my lists.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • We can use the classes we've made in HTML to make specific changes to their styles in CSS.
  • The connection from our HTML to CSS is typically put under last meta over the title.

Git

  • I've learned to use cd and pwd to change positions in git bash, and to clone and pull/push my code.
  • Git Cheat Sheet Link
  • Use git checkout -b (name) to create new branches, using git checkout normally can be used to move branches.
  • Typical workflow can look like: git add -A : git commit -m (quotes) : git push origin main : git push origin main
  • It is a good idea to constantly check git status to make sure our branch is where we want it to be.

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • We can create fucntions and call them to perform specific logic for our website.
  • The connection from our HTML to JavaScript is typically put under footer over the body closer.
